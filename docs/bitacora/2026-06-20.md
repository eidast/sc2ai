# 2026-06-20 — Bitácora

## Motor Estratégico Basado en Utilidad (Utility-Based Strategic Engine)

Implementamos un nuevo motor de decisiones estratégicas que reemplaza el sistema monolítico de FSM (`decision.py`) por un sistema de utility scoring continuo. El cambio se encuentra documentado en `openspec/changes/strategic-utility-engine/`.

### Motivación

El bot anterior tomaba decisiones mediante funciones deterministas sin contexto compartido. Cada `manage_*` operaba en aislamiento, sin noción de estrategia global ni capacidad de adaptación más allá de umbrales fijos. El nuevo sistema introduce un campo de fuerzas continuo donde cada acción posible recibe un score de utilidad calculado en tiempo real, y la mejor gana.

### Decisiones Clave

1. **Utility-Based AI**: En lugar de build orders rígidos o árboles de decisión, usamos un `PriorityEngine` que evalúa ~40 acciones cada step y selecciona la de mayor utilidad — el mismo enfoque usado por The Sims, Civilization, F.E.A.R. y Halo 3.

2. **Biases continuos**: Cada estrategia no es un camino fijo, sino una configuración de "campo gravitacional" con biases [0.0–1.0] que inclinan las decisiones. El scouting enemigo ajusta estos biases gradualmente (no dispara cambios bruscos), permitiendo transiciones suaves entre composiciones.

3. **Perfiles YAML como fuente de verdad**: Las estrategias se definen como datos, no como código. Cuatro perfiles iniciales para Protoss: `standard_macro`, `stargate_open`, `robo_open`, `fast_expand`.

4. **Scouting con decaimiento temporal**: La información de unidades enemigas pierde confianza exponencialmente con el tiempo, forzando re-scouting periódico y evitando decisiones basadas en datos obsoletos.

5. **FSM preservado como safety net**: Los estados DEFEND/SURRENDER/WON se mantienen como override de alto nivel. El PriorityEngine maneja las decisiones tácticas; el FSM actúa como red de seguridad.

### Técnicas y referencias

- **Utility AI**: Sistema usado en videojuegos AAA desde hace 20 años. Cada acción tiene una curva de respuesta (utility curve). Se evalúan todas cada frame y gana la de mayor score. Referencia: Dave Mark, "Behavioral Mathematics for Game AI" (2009).

- **Exponential Scout Decay**: `confidence(t) = exp(-decay_rate * Δt)`. La confianza en información scouteada decae exponencialmente. Parámetro `scout_decay_rate` configurable por perfil (default 0.05).

- **Bias Speed Smoothing**: Exponential Moving Average aplicado a cambios de bias. `bias[n] = bias[n-1] + α * (target - bias[n-1])` donde α = bias_speed. Valores bajos (0.1) producen cambios graduales y seguros; valores altos (0.8) producen cambios rápidos y reactivos.

- **Multi-Armed Bandit context**: Aunque no implementado aún en este cambio, la arquitectura está preparada para aprendizaje de parámetros. Un modelo ML futuro podrá entrenar biases y parámetros usando datos de partidas reales sin modificar la lógica de ejecución.
